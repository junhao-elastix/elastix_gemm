WHY THIS HAPPENS (Normal FP16 Behavior):
================================================================================

FP16 Format: [sign][5-bit exp][10-bit mantissa]

For SMALL values (e.g., 0.001):
  - Exponent is small (denormalized or very low exp)
  - Mantissa bits represent TINY increments
  - Example: LSB might represent 0.00000054
  - Small rounding difference → Large LSB count
  - BUT: Absolute error still tiny!

For LARGE values (e.g., 0.024):
  - Exponent is larger (normalized)
  - Mantissa bits represent LARGER increments  
  - Example: LSB might represent 0.000015
  - Same rounding difference → Small LSB count
