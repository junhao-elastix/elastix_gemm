◆ ACX_SDK_STATUS

enum ACX_SDK_STATUS

Status enumerators used by various SDK functions and structures to represent their return state or the outcome of operations.

Enumerator
ACX_SDK_STATUS_UNDEFINED 

The Current state is undefined or not known, default state for SDK_STATUS enum.

ACX_SDK_STATUS_OK 

The operation was successful or the state is valid.

ACX_SDK_STATUS_INDEX_OUT_RANGE 

The supplied index is beyond the range of the contents it is being applied to.

ACX_SDK_STATUS_SIZE_OUT_RANGE 

The supplied size is too large or too small for the contents of what it is being applied to.

ACX_SDK_STATUS_ADDR_OUT_RANGE 

The supplied address is beyond the legal range for the operation.

ACX_SDK_STATUS_CHAN_OUT_RANGE 

The supplied channel is beyond the legal range for the operation.

ACX_SDK_STATUS_ARG_NULL 

One or more supplied arguments was null.

ACX_SDK_STATUS_TARGET_NULL 

The target of the operation was null.

ACX_SDK_STATUS_BAD_MODE 

The target is set to the incorrect mode for the requested operation.

ACX_SDK_STATUS_ALREADY_EXISTS 

The item/data already exists in the context it is being applied to.

ACX_SDK_STATUS_NOT_MAPPED 

The target is not currently mapped within the current context.

ACX_SDK_STATUS_RUNNING 

The requested operation is already in progress. For function calls the returning function was not instructed to restart/override it.

ACX_SDK_STATUS_BAD_STATE 

A state error/mismatch occurred.

ACX_SDK_STATUS_ERROR 

Generic error.

ACX_SDK_STATUS_ENABLE 

Indicates the target is enabled.

ACX_SDK_STATUS_DISABLE 

Indicates the target is disabled.

ACX_SDK_STATUS_ALLOC_FAIL 

Indicates that the operation failed due to not being able to allocate the required resource.

ACX_SDK_STATUS_BAR_OPEN_FAIL 

Indicates that one or more BARs failed to be opened.

ACX_SDK_STATUS_ADDR_UNALIGNED 

Attempt to perform an unaligned MMIO operation.

ACX_SDK_STATUS_INVALID_HANDLE 

Supplied handle is NULL or fails a validation check.

ACX_SDK_STATUS_NOT_READY 

Indicates the target of the operation is not ready or busy.

ACX_SDK_STATUS_NO_INIT 

Indicates the target has not been initialized and is not in a known state.

ACX_SDK_STATUS_NOT_SUPPORTED 

Indicates the target is not supported by the operation.

ACX_SDK_STATUS_TIMEOUT 

Indicates that an operation took too long to resolve and was marked as timing out.

ACX_SDK_STATUS_CANCELLED 

Indicates that an operation was cancelled.

ACX_SDK_STATUS_MISSED_INTERRUPT 

Indicates that an interrupt was missed.

ACX_SDK_STATUS_NUM_STATUSES 

Number of status enums.

◆ acx_util_linear_map_add()

ACX_SDK_STATUS acx_util_linear_map_add ( ACX_util_linear_map  map,
char *  key,
void *  item 
)

Adds a new entry to an 'ACX_util_linear_map' structure.

Parameters
mapAn already allocated 'ACX_util_linear_map'
keyThe key to associate with the new item
itemThe item to associate with the new key
Returns
ACX_SDK_STATUS_OK on success ACX_SDK_STATUS_ALLOC_FAIL if the structure could not allocate more room for the new entry
Note
This function will first call acx_util_linear_map_find() and replace the item if a matching key already exists, making it linear time. If no new item is found then the new item will be the last free index in the map.

◆ acx_util_linear_map_delete()

ACX_SDK_STATUS acx_util_linear_map_delete ( ACX_util_linear_map  map,
char *  key 
)

Deletes am entry from an 'ACX_util_linear_map' structure.

Parameters
mapAn already allocated 'ACX_util_linear_map'
keyThe key for the entry to delete from the map
Returns
ACX_SDK_STATUS_OK on success ACX_SDK_STATUS_NOT_MAPPED if an entry with a matching key could not be found
Note
This function will first call acx_util_linear_map_find(). If an item is found then all items will be shifted to fill the whole in the map, keeping it densely packed and maintaining relative item ordering.
