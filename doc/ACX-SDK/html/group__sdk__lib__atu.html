◆ acx_atu_find_regions()

int acx_atu_find_regions ( ACX_DEV_PCIe_device *  device,
uint32_t  bar_num,
uint32_t  num_buf_elements,
ACX_ATU_region_context *  regions 
)

Fills the passed-in array with the context for all ATU regions with addresses that overlap the addresses of a specified BAR.

Parameters
devicePointer to an opened PCIe device.
bar_numWhich bar to find overlapping regions for.
num_buf_elementsNumber of elements that the passed-in regions buffer can hold.
regionsPointer to an array of ACX_ATU_region_context elements that will be filled with the overlapping ATU regions.
Returns
The number of ATU regions found to be overlapping with the given BAR. Returns -1 if num_buf_element is zero, the regions buffer is NULL, or if the ATU can't be read. The return of this function may be larger than num_buf_element if more regions are found than the region buffer can hold. In this case the region buffer will contain the first num_buf_element found.
Note
This function only checks for region address overlap. It does not check the function number of the regions. The user is responsible for filtering regions based on function number.

◆ acx_atu_get_base_addr()

uint64_t acx_atu_get_base_addr ( ACX_ATU_region_context *  atu_region)

Gets the base address of the specified ATU region in the host's address space.

Parameters
atu_regionPointer to an ACX_ATU_region_context structure to get the base address from.
Returns
Region's base address in the host's address space.
Note
The base address defines the start of an ATU region in the host's address space. The full region space is defined as the address range from base to limit. This function does not read from the device's ATU. It only reads from the passed-in context structure in host memory. It is the users responsibility to ensure that this structure is up to date with the ATU by using acx_atu_get_region_context() or acx_atu_get_context().

Gets the target address of the specified ATU region in the device's address space.

Parameters
atu_regionPointer to an ACX_ATU_region_context structure to get the target address from.
Returns
Region's target in the device's address space.
Note
The target address defines the start of an ATU region in the device's address space. The full region space is defined as the address range from the target to the target + (limit - base). This function does not read from the device's ATU. It only reads from the passed-in context structure in host memory. It is the users responsibility to ensure that this structure is up to date with the ATU by using acx_atu_get_region_context() or acx_atu_get_context().

Sets the base address of the specified ATU region in the host's address space.

Parameters
atu_regionPointer to an ACX_ATU_region_context structure to set the base address for.
base_addrThe base address to write into the ACX_ATU_region_context structure.
Note
The base address defines the start of an ATU region in the host's address space. The full region space is defined as the address range from base to limit. This function does not update the device's ATU. It only updates the context structure in host memory. Use acx_atu_config_region() to update the device's ATU after calling this function.
