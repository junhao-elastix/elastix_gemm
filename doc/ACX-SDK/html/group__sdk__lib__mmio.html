#define ACX_MMIO_TRACE_MANAGEMENT   (1 << 0)

This includes output from:

  • acx_mmio_open_hand_constant()
  • acx_mmio_close_hand()
  • acx_mmio_set_hand_trace_mask()

◆ acx_mmio_open_hand_constant()

ACX_SDK_STATUS acx_mmio_open_hand_constant ( ACX_DEV_PCIe_device *  device,
unsigned  bar,
uint64_t  offset,
uint64_t  length,
struct ACX_MMIO_attributes *  attributes,
ACX_MMIO_handle *  handle_p 
)

Create an MMIO constant handle.

An MMIO constant handle is bound to a specified BAR and offset when created

Parameters
[in]devicePointer to an opened PCIe device
[in]barThe BAR number for the created handle
[in]offsetThe base offset into the BAR for the handle
[in]lengthMaximum additional offset allowed for access functions on the handle
[in]attributesPointer to the attributes structure (NULL may be used to specify default attributes)
[out]handle_pPointer for storing MMIO handle created
Returns
ACX_SDK_STATUS_OK on success
ACX_SDK_STATUS_INDEX_OUT_RANGE if BAR number is out of range
ACX_SDK_STATUS_NOT_MAPPED if BAR is not mapped
ACX_SDK_STATUS_ADDR_OUT_RANGE if offset+length exceeds the size of the specified BAR
ACX_SDK_STATUS_ALLOC_FAIL if malloc() of handle data structure fails

◆ acx_mmio_read_bar_16()

ACX_SDK_STATUS acx_mmio_read_bar_16 ( ACX_DEV_PCIe_device *  device,
uint32_t  bar_num,
uint32_t  bar_offset,
uint16_t *  value 
)

Memroy-mapped IO read of 16-bit unsigned data from a PCIe device using a Base Address Register (BAR)

Parameters
devicePointer to an opened PCIe device whose BAR will be read from
bar_numThe BAR number to read from
bar_offsetThe offset into the BAR to read from
[out]valueA pointer to a 16-bit unsigned integer. If the read is succesful then the function will store the value here, 0 will be stored if the read operation fails.
Returns
An ACX_SDK_STATUS that indicates the status of the operation: 'ACX_SDK_STATUS_OK' if the read completed successfully. 'ACX_SDK_STATUS_INDEX_OUT_RANGE' if the 'bar_num' was out of range. 'ACX_SDK_STATUS_NOT_MAPPED' if the BAR pointed to by 'bar_num' is not mapped. 'ACX_SDK_STATUS_ADDR_OUT_RANGE' if the 'bar_offset' is greater than the BAR's size. 'ACX_SDK_STATUS_ARG_NULL' if the 'device' or 'value' pointer is NULL.

◆ acx_mmio_read_dbi()

ACX_SDK_STATUS acx_mmio_read_dbi ( ACX_DEV_PCIe_device *  device,
uint64_t  reg_addr,
uint32_t *  value 
)

Ease-of-use function for reading from a PCIe status register over the DBI interface.

Parameters
devicePointer to an opened PCIe device whose status register will be read from
reg_addrThe DBI address of the register to read from
[out]valueA pointer to a 32-bit unsigned integer. If the read is successful then the function will store the value here, 0 will be stored if the read operation fails.
Returns
An ACX_SDK_STATUS that indicates the status of the operation: 'ACX_SDK_STATUS_OK' if the read completed successfully. 'ACX_SDK_STATUS_NOT_MAPPED' if the device does not have a mapped DBI reader. 'ACX_SDK_STATUS_ARG_NULL' if the 'device' or 'value' pointer is NULL. 'ACX_SDK_STATUS_BAD_STATE' if the DBI reader block is not in a known good state, could indicate an issue with initialization 'ACX_SDK_STATUS_DISABLE' the DBI reader block is marked as disabled, and may not be safe to read from 'ACX_SDK_STATUS_ERROR' an unspecified error occurred when trying to read from the DBI, if this happens then the error will likely have originated in the driver

)

Set tracing level for an MMIO handle.

Parameters
[in]handleMMIO handle
[in]maskTracing bitmask enables:
  • bit 0: ACX_MMIO_TRACE_MANAGEMENT

  • bit 3: ACX_MMIO_TRACE_READS
