Detailed Description

Low level interface for PCIe DBI access.

Warning
This interface is normally intended for SDK library internal use (e.g. SDK Library ATU Interface or SDK Library DMA Interface)

typedef ACX_SDK_STATUS(* ACX_DBI_reader) (ACX_DEV_PCIe_device *device, ACX_IP_block *dbi_block, uint64_t reg_addr, uint32_t *value)

Generic function for reading from a register over the DBI interface.

Parameters
devicePointer to an opened PCIe device whose DBI register will be read from
dbi_blockPointer to an ACX_IP_block, which is the DBI IP block being read from
reg_addrThe DBI address of the register to read from
[out]valueA pointer to a 32-bit unsigned integer. If the read is successful then the function will store the read value here, 0 will be stored if the read operation fails.
Returns
An ACX_SDK_STATUS that indicates the status of the operation: 'ACX_SDK_STATUS_OK' if the read completed successfully. 'ACX_SDK_STATUS_ERROR' an unspecified error occurred when trying to read from the DBI, if this happens then the error will likely have originated in the driver.

◆ acx_dbi_gateway_overide_location()

ACX_SDK_STATUS acx_dbi_gateway_overide_location ( ACX_DEV_PCIe_device *  device,
int  compressed,
int  bar_index,
uint32_t  bar_offset 
)

Runtime override for the BAR index and offset for the gateway DBI interface.

Parameters
deviceA pointer to an open PCIe device
compressedA flag to indicate whether the compressed or full gateway should be modified. If 0 gateway will be in full mode.
bar_indexThe new BAR index for the gateway DBI interface
bar_offsetThe new BAR offset for the gateway DBI interface
Note
The default BAR location is statically defined in /src/Achronix_IP/Achronix_IP_config.c Depending on bitstream configuration it may be necessary to change the location during two stage programming. This function will call the IP block's initializer again if the block was not initialized when the device struct was created. In a future release these will be read in from a metadata file at runtime and this function will be deprecated.
