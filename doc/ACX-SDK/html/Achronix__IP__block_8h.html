Detailed Description

This file defines a generalized mechanism for software applications to interact with IP blocks (hard or soft IP) on a device.

By definition, soft-logic IP blocks ("soft IPs", or "blocks") programmed into an FPGA device cannot have a statically-defined interface. The block may or may not be present in the currently loaded bitstream, and the configuration/status registers it defines are application dependent.
In fact, multiple copies of a block might be present in the device. If, for example, the host application communicates with the block using the Network on Chip (NoC), the row and column address of the block's Network Access Points (NAPs) may change after an SDK application is compiled.
This file defines a generalized mechanism for the specification of a software interface for such soft IPs such that an SDK application can easily and portably interact with them.
