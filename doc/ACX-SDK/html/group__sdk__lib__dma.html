Detailed Description

SDK interface for DMA operations. This interface makes use of the SDK Library DBI Interface low level IP interface.

Enumeration Type Documentation

◆ ACX_DMA_DIRECTION

enum ACX_DMA_DIRECTION

Used to specify the direction data will flow during a transfer.

Enumerator
ACX_DMA_HOST_TO_DEVICE 

Transfer from host memory to device memory (a DMA "read" from the controller's perspective)

ACX_DMA_DEVICE_TO_HOST 

Transfer from device memory to host memory (a DMA "write" from the controller's perspective)

◆ ACX_DMA_LIST_STATE

enum ACX_DMA_LIST_STATE

Used to indicate the current state of a ACX_DMA_transaction structure when in the ACX_DMA_TACTN_LIST mode.

Enumerator
ACX_DMA_LIST_STATE_XFER_DESC 

Used to informa the config/transfer functions that the list node's descriptors need to be uploaded to the device.

ACX_DMA_LIST_STATE_XFER_LIST 

Used to informa the config/transfer functions that the list node's descriptors have already been uploaded to the device.

◆ ACX_DMA_TACTN_MODE

enum ACX_DMA_TACTN_MODE

Used to indicate the current mode of a ACX_DMA_transaction structure.

Enumerator
ACX_DMA_TACTN_UNDEFINED 

Default mode of a transaction after it has been allocated with acx_dma_alloc_tactn()

ACX_DMA_TACTN_BUFFER 

Used to transfer a single dma_buffer that has been allocated with acx_dma_alloc_buf()

ACX_DMA_TACTN_LIST 

Used to transfer one or more dma_buffers that are packed into an ACX_DMA_list_node object.

◆ ACX_DMA_TRANSFER_STATUS

enum ACX_DMA_TRANSFER_STATUS

Used to indicate the current status of a DMA transfer on a particular channel.

Enumerator
ACX_DMA_XFER_UNDEFINED 

Undefined status. Transaction status has not been set.

ACX_DMA_XFER_RESERVED 

Reserved state. Set when the channel is configured but not yet started.

ACX_DMA_XFER_RUNNING 

The channel is active and transferring data.

ACX_DMA_XFER_HALTED 

An error condition has been detected and the DMA has stopped this channel.

ACX_DMA_XFER_STOPPED 

The DMA has transferred all data for this channel, or it has been manually stopped.

ACX_DMA_XFER_COMPLETE 

Set when we have DMA_STOPPED and (ACX_DMA_CAP_TRANSFER_SIZE==0) in the _ACX_DMA_CHANNEL_REG register.

ACX_DMA_XFER_ERROR 

Indicates an error occurred and the transfer state is unknown.

◆ acx_dma_alloc_buf()

ACX_DMA_buffer* acx_dma_alloc_buf ( ACX_DEV_PCIe_device *  device,
uint64_t  size_in_bytes 
)

Allocates a DMA buffer in host memory.

Parameters
devicePointer to the device that will be associated with this buffer.
size_in_bytesNumber of bytes to allocate for the buffer.
Note
DMA buffers are handled differently from ordinary memory by the operating system, so one cannot simply use the malloc function to allocate a DMA buffer.
Warning
DMA buffers are backed 1:1 by physical host memory, this means that trying to allocate more memory than the host. physically has installed will likely crash the system or cause instability. This function will not check if the host has enough free memory to make the allocation.

Parameters

device_addressThe physical address that this node will be written to on the device
directionWhich direction the data should flow in (host-to-device or device-to-host).
Returns

Note
DMA buffers for the payload must be allocated separately using acx_dma_alloc_buf(). Use acx_dma_set_list_node_data_desc() to configure each of the descriptors in the node. The DMA buffer that holds the descriptors must be uploaded to the device before the descriptors can be used in a transaction. Care must be taken not to write over the descriptors during any subsequent data transfers.

Builds a passed-in ACX_DMA_transaction object by associating it with all state required to perform a single-buffer DMA transfer.

Parameters
transactionA pointer to the ACX_DMA_transaction object to be configured.
bufferA pointer to a pre-allocated ACX_DMA_buffer that will be held by the transaction object.
channelIndex of the DMA channel to use for the transfer.
directionWhich direction the data should flow in (host-to-device or device-to-host).
device_addressThe physical address on the device that the ACX_DMA_buffer will be read from or writen to.
buffer_offsetFor single-buffer (non-list) mode only, the offset into the physical address on the host that the ACX_DMA_buffer will be read from or writen to.
byte_countThe number of bytes to read/write from the dma_buffer
Returns
Status of the operation. ACX_SDK_STATUS_OK if successful
Note
This function does not modify the DMA channel context or DMA channel control registers, it only updates the passed-in transaction object in host memory. Use functions such as acx_dma_config_xfer and acx_dma_start_xfer to actually setup and run a DMA transfer.
Warning
This function does not cleanup or manage any previous objects held by the passed-in transaction. It is the users responsibility to de-allocate any objects that are being replaced.

◆ acx_dma_cleanup_list_node()

void acx_dma_cleanup_list_node ( ACX_DMA_list_node *  list_node)

De-allocates a descriptor list node buffer and de-allocates any associated ACX_DMA_buffers.

Parameters
list_nodePointer to the descriptor list node object to be de-allocated
Warning
The ACX_DMA_list_node tracks buffers on a per-descriptor basis. If two descriptors refer to the same buffer it will result in a double free when this function cleans them up. Buffer allocation is dependent on the used driver so this may or may not be undefined behavior. If a node has multiple descriptors that refer to a single buffer, a safer operation might be to free them manually and then use acx_dma_cleanup_list_node_no_payload() to clean up the node.

It allocates the needed source DMA buffer, creates a transaction object, configures the transfer, and starts a DMA transfer, all with one function call. The data in the passed-in buffer is copied into the transaction's DMA buffer before the DMA is started.
