Detailed Description

Holds all of the state needed for an application to interact with a soft or hard IP block on the device.

A soft or hard IP block presents an application-specific interface to the application software. The block may or may not be present on the device, and it may be enabled or disabled by the programmer. In general, all IP blocks are accessed through a BAR, so an application must know which BAR to use, and the address range (offset and limit) within that BAR. An IP block may or may not be initialized, and it may or may not be enabled. It may have an application-specific 6-bit status value, and it has a user-defined tag string. Finally, any other application-specific state may be placed in an arbitrary data-structure created by the initializer, and pointed to by the 'details' void pointer. These structures are populated at application startup time by the ACX_IP_initializer functions, and stored on the device struct in host memory. The tag field of the IP block is used to identify the block by the user. The tag field must be unique.
