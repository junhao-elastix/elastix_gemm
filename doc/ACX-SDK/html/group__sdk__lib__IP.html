◆ ACX_IP_cleaner

typedef void(* ACX_IP_cleaner) (ACX_DEV_PCIe_device *device, ACX_IP_block *ip_block)

Specification for cleanup functions that can be registered to be called at device cleanup time.

Each cleaner function is responsible for doing any cleanup that is required by the passed-in IP block. This includes freeing any structures that are stored in the details field that were setup by an initializer, or stopping block operation and placing it into a reset state.

Parameters
deviceA pointer to the device structure that is currently being cleaned up.

◆ ACX_IP_initializer

typedef uint32_t(* ACX_IP_initializer) (ACX_DEV_PCIe_device *device, ACX_IP_block *ip_block, void *details)

Specification for initialization functions that can be registered to be called at device initialization time.

Each initializer will be passed an IP block and device structure. Initializers are responsible for performing any required setup on the block they are passed. This could be things like placing the IP into a known state. One Initializer may be registered to multiple blocks and an initializer may be called multiple times with the same block if two-stage programming is used, so initializers must take care to check device/block state before performing any one-time setup. The index, offset, limit, and tag fields of a block will all be pre-initialized by the device initialization handler and should be considered read-only by initializers. The initializer may also create a custom structure within the IP implementation, used for any internal purpose, and point to it via the details field. The order of initialization is based on the order of function registration. However, it is recommended that each IP initializer does not rely on other blocks already being initialized. Initializers must return 0 for success. If an initializer experiences an unrecoverable error during setup, and requires device initialization to fail, then the initializer may return a non-0 value. A non-0 return will cause the device initialization handler to fail device initialization. Handling of recoverable errors is implementation defined. Each IP block has a 6-bit status field provided that an implementation may use to set statuses and errors.

Parameters
deviceA pointer to the device structure that is currently being initialized
ip_blockThe specific IP block to initialize
detailsReserved for future use
Returns
0 for success, non-0 for an error that requires device initialization to fail

◆ acx_ip_register_cleaner()

ACX_SDK_STATUS acx_ip_register_cleaner ( ACX_IP_cleaner  cleaner_function,
char *  block_tag 
)

Registers an IP block cleanup function that will be run when a device is closed.

Parameters
cleaner_functionACX_IP_cleaner function to be registered.
block_tagThe tag of the IP block to pass to the cleaner_function, if the block is present
Returns
Returns ACX_SDK_STATUS_OK if registration was successful, or ACX_SDK_STATUS_ALREADY_EXISTS if the function is already registered.
Note
Cleaner functions are run in the reverse order of registration. If {a, b, c} are registered in order then 'c' will be cleaned first and 'a' will be cleaned last. Cleaner functions may be registered to multiple block tags but may not be registered to a single tag more than once. Tag mappings are 1 to 1, a single tag cannot have multiple cleaners

◆ acx_ip_remove_cleaner()

ACX_SDK_STATUS acx_ip_remove_cleaner ( char *  block_tag)

Removes the tag mapping for a cleaner function.

Parameters
block_tagThe tag that is mapped to the cleaner
Returns
Returns ACX_SDK_STATUS_OK if removal was successful, or ACX_SDK_STATUS_NOT_MAPPED if the function was not found in the list of registered functions.
Note
The relative order of cleaners is preserved after the remove operation. For example if cleaners {a, b, c} are registered and 'b' is removed, the new cleaner order will be {a, c}
