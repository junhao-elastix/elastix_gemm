Table 232: Port Descriptions


Name

Direction

Description


rstn


Input

Asynchronous reset input. Resets the NAP interface but does not affect the NoC.


output_rstn


Output

Reset output from the NAP to the fabric logic. Do not use. Intended for use with partial reconfiguration. Signal is controlled by a write to the configuration space. Currently, the signal is fixed to 1'b0.


clk


Input

All operations are fully synchronous and occur upon the active edge of the clk

input.

tx_ready

Output

Asserted high when the NAP can accept data.

tx_valid

Input

Assert high to issue a word of data to the NAP.

tx_dest[3:0]

Input

The 4-bit destination ID of the row to which this word of data should be sent.


tx_sop tx_eop


Input

Start of packet and end of packet indicators. These values are sent unmodified to the destination. Use of these signals is optional. The purpose is to indicate to the receiving node the start and end of any packet transmission. The NAP (and NoC) do not require or make use of these signals. The tx_data word, tx_sop and tx_eop are transmitted on each tx_valid assertion.

tx_data[292:0]

Input

Data transmitted to the destination node.

rx_ready

Input

Asserted high by the user logic to indicate that it is ready to receive data.

rx_valid

Output

Asserted high with each valid rx_data word.

rx_src[3:0]

Output

The 4-bit transmission source ID indicating the row that originated the data.

rx_sop rx_eop


Output

Start of packet and end of packet indicators. These values are received unmodified from the source.

rx_data[292:0]

Output

Received data.
