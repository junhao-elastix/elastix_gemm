‌Usage and Inference

ACX_INT_RLB_MULT implements multiplication with reconfigurable logic (ACX_MLUT, ACX_ALU8 (see page 91), and ACX_DFF's (see page 24)), without using ACX_MLP72 (see page 95). This macro is similar to ACX_INT_MULT (see page 175) with the parameter architecture = "rlb".

There are two reasons to use ACX_INT_RLB_MULT:

it supports distinct sizes for 'a' and 'b' inputs, such as 5 Ã— 8 multiplication it supports 9-bit integers

However, ACX_INT_RLB_MULT only supports signed integers and does not support 16-bit and 32-bit multiplication.

Note

It is not currently possible to infer MLUT-based multipliers. ACX_INT_MULT (see page 175) or ACX_INT_RLB_MULT must be instantiated directly.

In addition to direct instantiation in Verilog or VHDL, an instance of ACX_INT_RLB_MULT can also be created in the ACE IP Configuration Perspective. See Speedster7t Soft IP User Guide (UG103) for details.
