‌Description

noc_delay

The Tcl command prints the round-trip delay when accessing a CSR register. This command confirms

that the ACX_DEVICE_MANAGER can communicate with the 2D NoC. However, its main purpose is to print whether the CSR clock is the system clock or the JTAG clock. For the ACX_DEVICE_MANAGER to operate correctly, the

use_acx_device_manager

set_csr_clock

system clock should be used. If the clock is reported as the JTAG clock, most likely the

command was omitted. The clock can be switched with the

use_acx_device_manager

command must also be used.
